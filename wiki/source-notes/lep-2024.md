# Rethinking Vector Embeddings Search for Analytical Database Systems (LEP)

Elena Krippner, Master's Thesis, TU Munich / CWI, 2024. Supervisors: Thomas Neumann, Peter Boncz, Leonardo Xavier Kuffo Rivero.

## Summary

LEP (Lossily Encoded floating-Points) adapts the ALP lossless float compression algorithm for lossy compression of vector embeddings. The key insight: ALP's per-vector encoding scheme (decimal→int or front-bits, operating on 1024-value batches) can be applied column-wise to embedding matrices. By adjusting the number of retained bits, LEP becomes a tunable lossy compressor achieving compression ratios comparable to PQ/SQ at similar recall levels, but with more accurate value reconstruction (lower MSE). The thesis also explores data layout effects (horizontal vs vertical), bitmap compression for frequent values, and dimension correlation exploitation.

## Key Contributions

- **ALP for embeddings**: Applies the ALP encoding scheme dimension-by-dimension to embedding matrices, creating a tunable lossy compression pipeline for vector search.
- **Data layout analysis**: Evaluates horizontal (row-major, one vector at a time) vs vertical (column-major, one dimension at a time) storage, showing vertical layout generally yields better compression due to per-dimension value regularity.
- **Bitmap compression of frequent values**: Identifies that some float values repeat frequently within a dimension; uses bitmap encoding for these common values.
- **Correlation-based compression**: Exploits Pearson correlation between dimensions to delta-encode one dimension relative to another.
- **Compression-recall Pareto analysis**: Shows LEP can match or exceed quantization-based methods (PQ, SQ) in the compression-recall space.

## Algorithm Details

### LEP Encoding (per dimension)
1. Sample a batch of values from one dimension across vectors.
2. Check if values are representable as scaled decimals (ALP's decimal detection).
3. If yes: encode as scaled integers with configurable bit width (lossy by truncating low bits).
4. If no: encode front bits only (sign + exponent + high mantissa bits), discarding low mantissa bits.
5. Store metadata (scale factor, bit width, exception flags) per dimension.

### Extensions
- **Bitmap encoding**: For dimensions with frequently repeated values, store a bitmap mask + frequent value dictionary.
- **Correlated dimension compression**: For dimension pairs with high Pearson r, store one as delta from the other.
- **PFOR exceptions**: Scalar values outside the encodable range are stored as exceptions.

## Reported Results

- Compression ratios comparable to or better than SQ and some PQ configurations at matched recall.
- Lower MSE than quantization-based methods at the same compression ratio.
- Vertical data layout consistently outperforms horizontal for compression.
- Encoding speed can be improved by sampling (reduced precision vs speed trade-off).

## Significance

LEP bridges the lossless float compression literature (Gorilla → ALP lineage) with vector embedding compression. It shows that ALP's decimal-aware encoding, originally designed for time series, transfers well to embedding dimensions when applied column-wise. This is one of the few works connecting float-level encoding techniques to ANN-scale vector compression.

## Limits And Open Questions

- Master's thesis — not peer-reviewed at a top venue. Results should be validated against mature implementations.
- LEP is lossy (unlike ALP) — the trade-off between bit truncation and recall needs per-dataset tuning.
- Does not exploit vector-level structure (only per-dimension and pairwise correlation) — PQ's subspace approach may capture richer patterns.
- Encoding time is higher than simple downcasting; the sampling speedup helps but reduces compression quality.
