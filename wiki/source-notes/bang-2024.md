# BANG: Billion-Scale Approximate Nearest Neighbour Search using a Single GPU

## Bibliographic Note

Karthik V., Saim Khan, Somesh Singh, Harsha Vardhan Simhadri, and Jyothi Vedurada. arXiv:2401.11324, v4 dated 2025-04-12. *BANG: Billion-Scale Approximate Nearest Neighbour Search using a Single GPU*.

Primary PDF: <https://arxiv.org/pdf/2401.11324>.

The system name is [BANG](../entities/bang.md).

## Core Problem

BANG targets graph-based ANNS when the dataset and graph index do not fit in one GPU. Prior GPU graph systems either keep all data and index structures in GPU memory or shard the dataset into GPU-sized pieces, creating heavy PCIe traffic and low throughput at billion scale.

The paper asks whether a single GPU can deliver high-throughput graph ANNS on billion-scale datasets without requiring all graph data to reside in GPU memory.

## Design

BANG combines CPU-resident graph access with GPU-resident compressed distance computation.

- **Graph on host memory:** the CPU keeps the graph index and retrieves neighbor lists on demand.
- **Compressed vectors on GPU:** product-quantized vectors fit in GPU memory and are used for approximate distance computation.
- **Phased execution:** split the ANNS workflow into kernels and CPU/GPU phases instead of one monolithic GPU search.
- **CPU-GPU pipelining:** overlap neighbor fetching, transfers, distance computation, sorting, worklist updates, and candidate prefetching.
- **Graph-index agnostic search:** the implementation uses Vamana, but the method is framed around proximity graphs more broadly.

## Evaluation Facts

- The headline setup uses one NVIDIA Ampere A100 GPU.
- At recall 0.9 on three popular billion-scale datasets, BANG reports 50x-400x higher throughput than competing methods.
- On SIFT1B and Deep1B at high recall 0.95, the paper reports 50.5x higher average throughput over competing methods.
- The paper reports peak GPU memory usage around 76 GB on SIFT1B/Deep1B, dominated by compressed vectors.
- BANG's optimizations include multi-kernel decomposition, prefetching candidate nodes, and asynchronous host-GPU transfers; disabling them reduces throughput in the paper's ablations.
- The compression ablation reports no noticeable recall or throughput drop on SIFT1B when reducing the PQ compression ratio down to 0.25, with degradation below that point.

## Relevance To Memory-Tiered ANN

BANG is a key single-GPU graph baseline for out-of-GPU-memory ANN. It is adjacent to [RUMMY](../entities/rummy.md) because both treat CPU-GPU transfer and scheduling as first-class problems, but BANG uses graph traversal and compressed vectors rather than IVF cluster processing.

It is also a comparison point for [GustANN](../entities/gustann.md), which argues for lower per-query GPU memory and SSD-aware traversal under high concurrency.
